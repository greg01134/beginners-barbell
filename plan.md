# Project Plan: The Beginner's Barbell

* Risk Analysis (TAME Framework)
Every web project carries potential risks. Below is an analysis of potential risks for this project and how they will be managed using the TAME framework:

**Transfer (Shift the risk to a third party):** * Risk: Hosting the website securely and ensuring it stays online.

Action: I will transfer this risk to GitHub by utilizing GitHub Pages for free, reliable hosting rather than attempting to configure and pay for my own web server.

**Accept (Acknowledge the risk and its consequences):**

Risk: Minor design inconsistencies across different web browsers (like Chrome vs. Safari).

Action: I will accept that my site may not look pixel-perfect identically on every single browser or older device. Given the scope of a beginner project, perfectly cross-browser compatible CSS is a lower priority than a functional, well-structured site.

**Mitigate (Reduce the likelihood or impact of the risk):**

Risk: Losing project files or code due to a computer crash.

Action: I will mitigate this risk by committing and pushing my code to my remote GitHub repository frequently at the end of every work session.

Risk: Copyright infringement regarding images of the lifts.

Action: I will mitigate this by strictly sourcing photos from Creative Commons/free-to-use platforms like Unsplash or Pexels.

**Eliminate (Remove the risk entirely by changing the plan):**

Risk: The project becoming too technically complex and missing the deadline due to broken interactive features.

Action: I will eliminate this risk by strictly adhering to the scope.md exclusions and utilizing zero JavaScript. The site will rely purely on semantic HTML and CSS.
